- 認知心理学
    - [[Tversky]] の “[[Features of Similarity]]”
        - [[feature matching]] / [[contrast model]] を提案
- [[DeepWalk]] や [[node2vec]] のようなグラフ埋め込み
    - 有用な[[帰納バイアス]]
- ComplEx は、実数ベクトルの単純な内積では扱いにくい[[反対称関係]]を、複素ベクトルの[[エルミート積]]により扱える
- 点ではなく箱・領域として概念を表す [[Box Embedding]] 系
    - 「概念は一点ではなく領域である」「関係は距離ではなく包含・重なり・排他・条件付き確率である」
- 異なる種類の関係を、無ラベルのエッジや単一の類似度に射影すると、関係の合成可能性が失われる。そのうえでベクトル空間の近さとして扱うと、元の関係構造に存在しない連続性・三角不等式・クラスタ性を仮定してしまう。
    - 「[[関係がある]]」を「[[似ている]]」に潰すのが危ない。
    - 「異なる関係」を「同じ種類のエッジ」に潰すのも危ない。
    - さらにそれを[[ベクトル空間]]上の近さとして読むと、元の構造にはなかった[[推移性]]めいたものが混入する。

ジェンドリン<img src='https://scrapbox.io/api/pages/nishio/nishio/icon' alt='nishio.icon' height="19.5"/>
- 「[[関係ラベル]]」は [[handle]] に近い<img src='https://scrapbox.io/api/pages/nishio/GPT5/icon' alt='GPT5.icon' height="19.5"/>
- 「類似度」は、暗黙の複雑さを最低共通項に落としやすい
    - [[The New Phenomenology of Carrying Forward]]
        - [[Carrying Forward]] / [[crossing]]
- [[Thinking at the Edge]] 解説では、crossing は他の理論や言葉を自分の felt sense の中で暗黙に働かせ、自分の言葉の精密さを保ったまま、さらに言えるようにする過程として説明
- R1 ∘ R2 は、この文脈で何を [[carry forward]] するか
- A・B・C という点、R という関係、類似度という数値は、すべて[[生きた相互作用]]から後で切り出された形式である。形式化は必要だが、その形式が元の暗黙の複雑さを carry forward するか、それとも最低共通項へ flatten するかが分かれ目である。

川喜田二郎<img src='https://scrapbox.io/api/pages/nishio/nishio/icon' alt='nishio.icon' height="19.5"/>
- A–B、B–C、しかし A–C なし、という関係の繊細さを、単一の「近さ」に押し込むことは、KJ法でいう「混沌をして語らしめる」前に、こちら側の分類枠を先にかぶせてしまうことに近い。<img src='https://scrapbox.io/api/pages/nishio/GPT5/icon' alt='GPT5.icon' height="19.5"/>
- B が一つのラベルとして適切に単位化されているか
    - 分割されるべきかもしれない
    - 関係の文脈を持った B を、文脈抜きの単一 B にしてしまうことも問題
- KJ法では、仮説は最後に立ち上がる。
    - 粗い埋め込みでは、座標系そのものがすでに仮説を含んでいます。
- [[W型問題解決モデル]]から見ると、ベクトル化は「思考レベル」への早すぎる移行
- 単なるエッジとして処理しない
    - A と B は、どの場面で、何によって関係したのか
    - B と C は、どの場面で、何によって関係したのか
    - A と C は、なぜ関係しないと言えるのか

Cosense<img src='https://scrapbox.io/api/pages/nishio/nishio/icon' alt='nishio.icon' height="19.5"/>
- Cosense は、まさに「関係ラベルを消して、単なるリンクにしている」道具<img src='https://scrapbox.io/api/pages/nishio/GPT5/icon' alt='GPT5.icon' height="19.5"/>
    - Cosense はそのリンクを 類似度 や 推論規則 にまで自動変換しない
    - Cosense は A–B–C の 2-hop 関係を見せますが、それを A と C には関係がある とは断定しません。
    - A から見たとき、B を介して C が見える という [[発見の候補]] として提示する。
        - Cosense の 2-hop は「推論」ではなく「[[セレンディピティ]]」
- Cosense は関係ラベルをリンクに潰している。
    - しかし、Cosense のリンクは孤立したエッジではなく、本文の中に埋まったリンクです。
    - 関係ラベルはエッジ型としては保存されていなくても、周辺テキストに残ります。
        - リンク周辺の自然言語が関係ラベルの役割を担う
- KJ 法では、最初から分類しない。Cosense でも、最初から階層分類しない。
    - KJ 法は、ある時点でグループ編成、表札づくり、図解化、叙述化へ進む。
    - Cosense は、その統合フェーズを強制しません。
    - だから Cosense は、KJ 法のような「発想の場」を常時開きっぱなしにする道具に近い。
- ジェンドリンに照らすなら、Cosense の `[B]` は概念の確定ではなく、felt sense への handle に近い。
    - [[すべての Wiki は書きかけ]] = すべてのページはfelt senseへのハンドル<img src='https://scrapbox.io/api/pages/nishio/nishio/icon' alt='nishio.icon' height="19.5"/>
        - [/rakusai/すべてのWikiは書きかけ](https://scrapbox.io/rakusai/すべてのWikiは書きかけ)
- Scrapbox/Cosense は「点」より「ページ」を基本単位にする
- 前の議論では、「関係ラベルを消して単なる類似や無ラベルエッジにするのが危ない」と言いました。Cosense に照らすと、これは少し修正できます。
    - 関係ラベルを必ず形式的なエッジ型として保存せよ、ではない。
    - ただし、[[関係の質を後から読める形で残せ]]。
