*1210396076*LinCity クリア
<a href="http://www.google.co.jp/codesearch?hl=ja&q=+package:http://ftp5.ru.freebsd.org/pub/FreeBSD/distfiles/lincity-ng-1.0.3.tar.bz2+UNIVERSITY_TECH_MADE+show:2Wblys_OL8U:NdQZGanPM6c:c7g2H0GDAJI&sa=N&cd=1&ct=rc&cs_p=http://ftp5.ru.freebsd.org/pub/FreeBSD/distfiles/lincity-ng-1.0.3.tar.bz2&cs_f=lincity-ng-1.0.3/src/lincity/lin-city.h">ソース</a>を読んでみた。ヘルプの解説に反して、大学を作るよりも学校をたくさん作る方が効率が良さそうだと思った。

ソースを読んでわかったこと。ホームレスの増加は失業とは無関係。単に住居が足りないだけ。ホームレスは何も生産せずに資源を取っていく。ヘルプでは炭焼き場(Commune)が1つあれば製陶所(Pottery)をいくつもまかなえると書いてあったけども、communeはcoal +2, ore +6でpotteryはcoal -1, ore -11, goods +35だ。足りてない。農場がフルに稼働すると(技術力ボーナスと電力ボーナスをのぞいて) food +450。これは養える人口でもある。ただ、就業人口は全「家に住んでいる」人口の45%。ホームレスが働いているかどうかはちゃんと読んでないけど働いてないと思う。

でまぁ、最終的に必要になるのは技術レベルを上げるための学校(技術レベルがロケットの打ち上げ成功率にもなるので100%にキープしたい)に必要なJob, Goodsと、ロケットを作るのに必要なJob(労働力), Goods(加工品), Steel(鉄鋼)、そして潤沢な資金だ。ロケットは作っている最中でも維持費が掛かるのでなるべく速やかに完成させたい。そのためには材料を潤沢に用意したい。鉄鋼を生産するには製鉄所(重工業)が必要で、製鉄所にはたくさんの石炭と鉱石を供給する必要があり、Goodsを生産するには工場(軽工業)が必要で、これは鉱石と鉄鋼が必要で、鉱石を得るには・・・そしてそのすべてに労働力が必要。これをロケット周辺で用意して遅滞なく輸送するのは大変だよね。なので「ロケットを港のそばに作ってSteelとGoodsは輸入する」という選択肢をえらんだ。お金さえ潤沢にあればこれで気にすることは労働力の供給だけになる。

この「労働力の供給」もあることに気づくまではだいぶ手間取ったのだけど、ようするに「ロケットを作り始めるのには技術力が必要」「ロケット打ち上げ時には高い技術力が必要」と制約を満たすのに「ロケットを作っている最中も技術力をキープすることが必要」ではない。ロケットを作り始めるやいなや周辺の学校を破壊して回ってそこで浮いた労働力をロケット生産につぎ込んだ。あと、ロケットができ次第どんどん飛ばしていたんだけど、次のロケットを作っている間に人口が増えてしまうので効率が悪い。維持費を払ってでもロケットをためておいて、人口を超えてからまとめてとばす必要がある。ロケットをためることに気づいたのも学校を壊す選択肢にたどり着いた一因か。まえはロケットができ次第飛ばしていたからロケットが完成する時期に技術力を高くしておく必要があったわけ。すぐに飛ばすのをやめさえすればその縛りがなくなる。

なお、あっさりクリアできているように見えるけどもこの過程で700ミリオンあった資金が200ミリオンになってしまったのでよほど資金に余裕がないとできないかも。でも資金に余裕がない状態で一機2ミリオンとかするロケットを十数機もつくれないよね。

とまぁそんなわけで「全人口をロケットで旅立たせる」クリア。もう一つのクリア方法の「Sustainableな経済を築け」は難しそうだ。

>>
 Processing.js 概要

まず、 Processing.js とは何かという話をします。

Processing.js とは、ブラウザで Processing というプログラミング言語を実行する JavaScript のライブラリです。

では、 Processing とはどのようなプログラミング言語なのでしょうか。

Processing とは

Processing とは、ビジュアルデザインのためのプログラミング言語で、 Java のような文法をしています。

ビジュアルデザインのためのプログラミング言語というと難しそうですが、実際はとても簡単です。
<<

これを読んだ読者は「ProcessingというJavaで書かれたビジュアルデザインのための言語(&統合開発環境)があり、ブラウザ上のJSでも可能な描画まわりの機能を移植したのがProcessing.jsだ」と理解するだろうか。それとも「ProcessingっていうすごいJSのライブラリができたらしい。これを使えばこんな絵が簡単に描けるのか、よしProcessingを勉強しよう」と言ってProcessing.jsを使い始めるだろうか。

Processing.jsがProcessingの一部の機能を移植しただけに過ぎない(そして一部の機能は移植が困難だったりする)ことに言及するか、せめてWikipediaにリンクするとかしてもいいんじゃなかろうか。
