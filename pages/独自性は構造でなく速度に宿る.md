- 「独自性は構造でなく速度に宿る」とは何か
    - 「LLM WikiがCosenseから継承したもの」(2026-05-22 articulate) が起点のテーゼ。本wiki の自己理解の中核命題のひとつ。
- 1 文要約
    - LLM Wiki の「新しさ」は、新しい知識構造を発明したことではなく、既にあった構造を安価に・高速に運用可能にしたことに宿る ([[人間と LLM のコスト逆転]] の系譜)。
- 起点
    - Cosenseフォーラムでの[[トランスクルージョン]]についての議論での「比較ページを作るとよさそう」という発言が、本wiki の analyses/ ディレクトリと同型のパターンだと nishio が気づいた。そこから自己理解の見直し:
        - 本wiki が「LLM Wiki の新しいパターン」として書いていた多くは、実は Cosense コミュニティが実践していた対処 を継承していた
        - →「LLM Wiki = Karpathy の新発明」ではなく「Cosense 実践者の文化 からの転写 + AI 速度」と解釈できる。

:
| 比較ページ作成 | 高 → 稀にしか書かない | LLM が ad-hoc に生成、日常的 |
| -- | -- | -- |
| view 合成 | 手書き → 稀 | LLM がクエリごとに生成 |
| 切り出し判断 | 経験と感覚 | 複合シグナル + LLM 提案 |
| 関連ページ発見 | 目視 | LLM が cross-ref を補完 |
- 構造は同じだが、各操作のコストが桁違いに下がった。Cosense 時代に「やれば便利だが高コストだから頻度が低い」操作が「日常的に回る」操作に昇格した。
