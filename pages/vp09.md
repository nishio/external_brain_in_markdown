<img src='https://scrapbox.io/api/pages/nishio/o3/icon' alt='o3.icon' height="19.5"/>**“vp09” は Google が開発したオープンかつロイヤリティフリーの動画コーデック **「VP9」を、MP4 などの ISO-BMFF コンテナ内で表す FourCC（4 文字コード） です。YouTube や TikTok 向けに yt-dlp などでフォーマット一覧を表示すると、vp09 と出ている行は「このストリームは VP9 で圧縮されている」という意味です。

<img src='https://scrapbox.io/api/pages/nishio/o3/icon' alt='o3.icon' height="19.5"/>
映像トラックが VP9（vp09）でエンコードされている
- Premiere Pro は 2025 年 7 月時点でも VP9 を「正式対応」しておらず、VP9 を含む MP4 を読み込むと 音声（AAC／Opus）だけを認識し、映像は「未対応コーデック」扱いで無視 されます。

yt-dlp なら、VP9 を避けて avc1（H.264）を指定すると Premiere でそのまま開けます。
:

```
# 例：最高解像度の H.264＋AAC を mp4 に結合
yt-dlp -f "bv*[vcodec=avc1]+ba*[acodec=mp4a]/best[ext=mp4][vcodec=avc1]" \
       --merge-output-format mp4 https://youtu.be/GNCHe0sYXJ4
```

ポイント
- VP9 を除外する	vcodec=avc1 または vcodec!=vp9
- 映像+音声を一つに	bv*+ba* ＋ --merge-output-format mp4
- -F で一覧を出し、avc1 が付いている ID を手動で組み合わせても OK です。

既に VP9 で落としてしまったファイルを変換
VP9 → H.264/ProRes など、Premiere が読めるコーデックに再エンコードします。

:

```
# 画質を保ちつつ再エンコード（例：H.264 高品質）
ffmpeg -i input_vp9.mp4 -c:v libx264 -crf 18 -preset slow -c:a copy fixed.mp4
```



> 手動で組み合わせても OK
なぜ組み合わせる？<img src='https://scrapbox.io/api/pages/nishio/nishio/icon' alt='nishio.icon' height="19.5"/>

動画＋音声が一体になった“プログレッシブ”形式（例: 18＝360p、22＝720p）がそもそも含まれていません。
- video only の DASH 映像ストリーム
- audio only の音声ストリーム
だけです。
したがって 映像と音声を「組み合わせる（mux する）」工程は必須 になります。

ユーザーが別途 ffmpeg を叩く必要はありません。たとえば次の 1 行で完結します。
:

```
yt-dlp -f "bv*[vcodec=avc1]+ba*[acodec=mp4a]" \
      --merge-output-format mp4 https://youtu.be/GNCHe0sYXJ4
```


- bv*+ba* と書けば 最適な映像＋最適な音声 を自動ペアリング
- --merge-output-format mp4 で最終ファイルを MP4 に統一
- vcodec=avc1 を付けて VP9 を除外すれば Premiere でもそのまま読める
