- Cosense の `[KJ法]` は recall・注意・navigation の3仕事を1文字列で兼ねている。だから recall のためにリンクせざるを得ず、その結果 490-page hub に膨れて注意が希釈する。grasp-最適の核は、この3仕事を層に分離して recall を link graph の外へ出すこと。recall が link から外れた瞬間、link は infrastructure(必須)から editorial(任意の表明)に変わる。
- link が recall を運ばないなら、KJ法 が 490 page で言及されているのは「壊れた link graph」ではなく「豊かな corpus」だ。
