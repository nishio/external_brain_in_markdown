"dogfooding loop" — 私の意図
dogfooding = IT 業界の jargon「自社製品を自分で使う」(eat your own dog food)。私の使い方は 「workflow を articulate した直後にその workflow を自分で実行する」。
loop をつけたのは 「各 session の output が次 session の入力 (seed) になる連鎖」 を指す。

具体的に「10 連続」が何だったか:

:

```
[1] 5/30 俯瞰 → 表札 A 生成
     ↓ output: 残り表札の seed 一覧
[2] 5/31 表札 C 生成
     ↓ output: 「2 表札の経験を articulate したい」
[3] 5/31 hyousatsu-experience articulate (10 気づき)
     ↓ output: 「気づきの 1 つを Q&A 化」/「違和感 trigger を articulate したい」
[4] 5/31 discomfort-trigger-bidirectional
     ↓ output: 「『継承』handle が不明確」 user 違和感
[5] 5/31 同一作者のwikiは親子でなく兄弟 (A/B/C typology)
     ↓ output: 「周期 session を articulate したい」
[6] 6/01 overview-triage-session
     ↓ output: 「CLAUDE.md に first-class 化」
[7] 6/01 CLAUDE.md 改訂
     ↓ output: 「残り表札 B/D/E/F」
[8-10] 6/01 表札 B → D → E1+E2+E3 → F (sub-pattern 4 個 articulate)
```

→ ポイントは 「workflow を articulate しながら同時に運用する」 こと。前 session の workflow をそのまま使うのでなく refine しながら使う。だから "loop" は単純反復ではなく螺旋に近い。

handle として失敗している点:
- "dogfooding" は本wikiで articulate されていない外来 jargon
- "loop" は「同じことを繰り返す」とも読めて進歩 nuance が抜ける
本wiki 流に言い直すなら: 「[[articulate と運用の交互 refine]]」 または 「[[workflow が運用の中で進化する pattern]]」
