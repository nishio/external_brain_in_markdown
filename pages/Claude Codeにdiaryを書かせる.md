[[Claude Code]]に[[diary]]を書かせる
- [[日報]]

実例は[docs/diaryで検索](https://scrapbox.io/nishio/search/page?search=docs%2Fdiary)すると見つかる

CLAUDE.md

```markdown
### LOG.md
- そのリポジトリでの**作業ログ**を書く場所。
- 実行中のエージェントがなんらかの事故によって急に終了した場合に、どこまで作業をしたのかを思い出すためのものである。
- 実行したコマンド・試したこと・結果などを**時系列に詳細に**残す。
- AIは、コードや設定を変更したときは**必ず LOG.md に記録する。**

### PLAN.md
- **「今進行中の計画」を1枚にまとめるファイル。**
- ToDoリストではなく、「このプロジェクトで今どこに向かっているか」を整理する。
- 計画が終わったら、各リポジトリのルールに従って
  - `docs/diary/completed_plans/` などに移動するか、
  - diaryにまとめを残してからアーカイブする。

### docs/diary/*.md
- 1日または1セッションの**作業のまとまった振り返り**を書く場所。
  - 例: `docs/diary/2025-11-25.md`
- LOG.md が「リアルタイム」「逐一ログ」だとすると、
  - diary は**1日の終わりに書く要約・振り返り**。
- AIが自動で書く場合も、**人間が読んで意味が通る文章**にする。

### CLAUDE.md
- そのリポジトリ固有の**「毎回覚えておくべき知識」**を書く場所。

### docs/*.md
- そのリポジトリ固有の「毎回覚えておくべきというほどでもないが繰り返し参照することが必要になりそうな知識」を書く場所。
- 書いたことを忘れては意味がないので、どこに何を書いたかをCLAUDE.mdに書く。
```
